# CS-360

AP Warehouse Inventory was built to give small warehouse businesses and individuals a simple way to track their stock. The app lets users create an account, log in securely, and manage inventory items through a clean grid display. Users can add, edit, and delete items, and receive SMS notifications when inventory is running low. The five screens — Login, Inventory Grid, Add Item, Item Details, and SMS Permissions — were each designed with a specific job in mind, keeping the layout intuitive and consistent throughout with clear labels, a purple color theme, and straightforward navigation.

I built the app one piece at a time starting with the UI, then the SQLite database, then connecting everything together through Java. Keeping the database logic in its own DatabaseHelper.java class and using consistent naming conventions made the code easier to follow and update as the project grew. I tested each feature in the Android Emulator as it was built, which helped catch issues early — like the inventory grid not refreshing after deleting an item from the details screen — before they became bigger problems. The part of the project I feel strongest about is the database layer, which handles all four CRUD operations cleanly and would be easy for anyone else to read and build on.
